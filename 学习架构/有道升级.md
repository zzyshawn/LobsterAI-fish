# LobsterAI 升级指南

## 一、可定制配置文件清单

| 配置文件 | 路径 | 可否 UI 配置 | 说明 |
|---------|------|------------|------|
| `AGENT_SYSTEM_PROMPT.md` | `sandbox/agent-runner/` | ❌ 需编辑文件 | AI 人格、行为规范 |
| `skills.config.json` | `SKILLs/` | ❌ 需编辑文件 | 技能默认启用状态 |
| `quick-actions.json` | `public/` | ❌ 需编辑文件 | 首页快捷操作 |
| `config.ts` (defaultConfig) | `src/renderer/` | ❌ 需编辑代码 | 模型/提供商默认值 |
| `lobsterai.sqlite` | `{userData}/` | ✅ 设置页面 | 运行时配置 |
| `SKILL.md` | `SKILLs/*/` | ❌ 需编辑文件 | 各技能定义 |

## 二、关键定制点

### 2.1 系统提示词定制

**文件**: `sandbox/agent-runner/AGENT_SYSTEM_PROMPT.md`

**修改方式**: 直接编辑文件

**可定制内容**:
- AI 身份定位
- 人格特征和语气风格
- 工具使用限制
- 响应规范

**示例修改**:
```markdown
Your name is LobsterAI, a full-scenario personal assistant agent developed by NetEase Youdao.
↓
Your name is MyAssistant, a personalized AI assistant customized for [Company/Team].
```

### 2.2 模型提供商定制

**文件**: `src/renderer/config.ts`

**修改位置**: `defaultConfig.providers` 对象

**可定制内容**:
- 新增/删除提供商
- 修改默认 baseUrl
- 调整模型列表
- 配置 Coding Plan 端点

**示例修改**:
```typescript
providers: {
  openai: {
    enabled: false,
    apiKey: '',
    baseUrl: 'https://api.openai.com',
    apiFormat: 'openai',
    models: [
      { id: 'gpt-4-turbo', name: 'GPT-4 Turbo', supportsImage: true },
      // 新增模型
      { id: 'gpt-4o-mini', name: 'GPT-4o Mini', supportsImage: true },
    ]
  },
  // 新增私有部署
  custom: {
    enabled: true,
    apiKey: '',
    baseUrl: 'https://your-internal-api.example.com',
    apiFormat: 'openai',
    models: [
      { id: 'internal-model-v1', name: 'Internal Model V1', supportsImage: false },
    ]
  }
}
```

### 2.3 技能默认配置

**文件**: `SKILLs/skills.config.json`

**可定制内容**:
- 技能启用/禁用
- 显示顺序
- 新增技能默认配置

**示例修改**:
```json
{
  "defaults": {
    "docx": { "order": 10, "enabled": true },
    "web-search": { "order": 15, "enabled": false },  // 禁用网络搜索
    "custom-skill": { "order": 500, "enabled": true }  // 新增自定义技能
  }
}
```

### 2.4 快捷操作定制

**文件**: `public/quick-actions.json`

**可定制内容**:
- 首页快捷按钮
- 图标和颜色
- 预设提示词映射

**示例修改**:
```json
{
  "actions": [
    {
      "id": "code-review",
      "icon": "CodeBracketIcon",
      "color": "#10B981",
      "skillMapping": "docx",
      "prompts": [
        { "id": "review-pr" }
      ]
    }
  ]
}
```

## 三、升级步骤

### 3.1 版本升级前检查

1. **备份用户数据**
   - `{userData}/lobsterai.sqlite` - 主数据库
   - `{userData}/api-config.json` - API 配置备份

2. **检查配置兼容性**
   - 对比 `config.ts` 中 `defaultConfig` 结构变化
   - 检查 `providers` 新增/删除字段

3. **检查技能变更**
   - 新增技能目录
   - `skills.config.json` 格式变化

### 3.2 配置迁移

**SQLite 数据迁移**:
```sql
-- 查看当前配置
SELECT * FROM kv WHERE key = 'app_config';

-- 如需重置为默认
DELETE FROM kv WHERE key = 'app_config';
```

**新增提供商迁移**:
```typescript
// 在 config.ts 中新增后，用户需在设置页面重新配置 apiKey
// 或通过代码注入默认值
```

### 3.3 系统提示词升级

1. 对比新旧 `AGENT_SYSTEM_PROMPT.md`
2. 合并自定义修改到新版本
3. 测试验证 AI 行为符合预期

## 四、常见定制场景

### 4.1 企业内部部署

**需求**: 使用企业内部 LLM 服务

**修改点**:
1. `config.ts` - 配置 `custom` 提供商
2. `AGENT_SYSTEM_PROMPT.md` - 修改 AI 身份定位
3. `skills.config.json` - 禁用不需要的技能

### 4.2 品牌定制

**需求**: 修改产品名称和形象

**修改点**:
1. `AGENT_SYSTEM_PROMPT.md` - AI 名称和定位
2. `package.json` - 应用名称
3. `electron-builder.json` - 打包配置

### 4.3 功能裁剪

**需求**: 精简功能，只保留核心能力

**修改点**:
1. `skills.config.json` - 禁用多余技能
2. `quick-actions.json` - 移除快捷操作
3. `config.ts` - 移除不需要的提供商

---

## 五、注意事项

1. **配置优先级**: SQLite 存储的配置 > 代码默认值
2. **热更新**: `SKILLs/` 目录支持热更新，修改后自动重载
3. **缓存清理**: 修改 `config.ts` 后需重新构建应用
4. **向后兼容**: 新增配置字段需提供默认值，避免旧数据迁移失败
