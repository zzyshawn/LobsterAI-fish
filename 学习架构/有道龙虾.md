# LobsterAI（有道龙虾）启动流程与配置文件分析

## 一、项目概述

LobsterAI 是网易有道开源的本地桌面 AI 智能助理，基于 **Electron + React + TypeScript** 构建。

**技术栈**: Electron 40 + React 18 + Redux Toolkit + Vite 5 + sql.js (SQLite WASM)

---

## 二、启动流程摘要

```
Electron 启动
    │
    ├─► 配置用户数据目录
    │
    ├─► 初始化日志
    │
    └─► app.whenReady()
          │
          ├─► 创建默认项目目录 (~/lobsterai/project)
          │
          ├─► 初始化 SQLite 数据库
          │
          ├─► 初始化技能管理器
          │     └─► 同步内置技能 + 监听目录变化
          │
          ├─► 启动技能服务
          │
          ├─► 启动 OpenAI 兼容代理
          │
          ├─► 创建主窗口
          │
          ├─► 创建系统托盘
          │
          ├─► 启动定时任务调度器
          │
          └─► 自动连接 IM 网关
```

## 四、核心配置详解

### 4.1 SQLite 数据库 (`lobsterai.sqlite`)

**加载入口**: `src/main/sqliteStore.ts`

**主要数据表**:
- `kv` - 通用键值存储（存储 `app_config`）
- `cowork_sessions` / `cowork_messages` - 会话和消息
- `user_memories` - 用户记忆
- `scheduled_tasks` - 定时任务
- `im_config` - IM 网关配置（钉钉/飞书/Telegram/Discord/网易云信）

### 4.2 应用配置 (`app_config`)

**存储**: SQLite `kv` 表，键名 `app_config`

**主要内容**:
- API 配置
- 多模型提供商配置（OpenAI/DeepSeek/Qwen/GLM/Anthropic/Gemini/Ollama 等）
- 主题、语言、代理设置
- 快捷键配置

**加载流程**:
1. 渲染进程: `ConfigService.init()` 从 SQLite 读取
2. 主进程: `resolveCurrentApiConfig()` 解析当前生效的 API 配置

### 4.3 模型配置 (`model` & `providers`)

**配置定义**: `src/renderer/config.ts`

**存储位置**: SQLite `kv` 表，键名 `app_config`

**配置结构**:
```typescript
{
  model: {
    availableModels: [...],    // 可用模型列表
    defaultModel: 'deepseek-chat'  // 默认模型
  },
  providers: {
    openai: { enabled, apiKey, baseUrl, apiFormat, models[] },
    deepseek: { enabled, apiKey, baseUrl, apiFormat, models[] },
    qwen: { enabled, apiKey, baseUrl, apiFormat, codingPlanEnabled, models[] },
    // ... 其他提供商
  }
}
```

**内置提供商** (共 13 个):

| 国内 (CHINA_PROVIDERS) | 国际 (GLOBAL_PROVIDERS) |
|----------------------|------------------------|
| deepseek | openai |
| moonshot | gemini |
| qwen | anthropic |
| zhipu | openrouter |
| minimax | |
| xiaomi | |
| volcengine | |
| ollama | |
| custom | |

> ⚠️ **注意**: 模型配置**没有单独配置文件**，是 `app_config` 的一部分。默认值硬编码在 `config.ts` 的 `defaultConfig` 中，用户在设置页面修改后存入 SQLite。

### 4.4 技能配置 (`skills.config.json`)

定义各技能的默认启用状态和显示顺序：

```json
{
  "defaults": {
    "docx": { "order": 10, "enabled": true },
    "web-search": { "order": 15, "enabled": true },
    "pptx": { "order": 30, "enabled": true }
  }
}
```

### 4.4 技能定义 (`SKILL.md`)

每个技能目录下的定义文件，采用 YAML Frontmatter 格式：

```markdown
---
name: PPT Generator
description: Generate PowerPoint presentations
---

技能说明文档...
```

## 六、配置加载顺序

```
1. 应用启动
2. configureUserDataPath() → 设置 userData 目录
3. SqliteStore.create() → 加载/创建 lobsterai.sqlite
4. SkillManager 初始化 → 读取 skills.config.json
5. 渲染进程 ConfigService.init() → 读取 app_config
6. resolveCurrentApiConfig() → 解析生效的 API 配置
7. 各 Store 按需加载对应配置表
```

## 七、新建任务流程

用户在前端点击"新建任务"后的完整执行链路：

```
前端发起请求
    │
    └─► IPC: cowork:session:start
          │
          └─► main.ts IPC Handler
                │
                ├─► 检查工作目录
                │
                ├─► 生成任务标题 (从 prompt 第一行截取)
                │
                ├─► CoworkStore.createSession()
                │     └─► INSERT INTO cowork_sessions
                │     └─► 返回 Session 对象 (status='idle')
                │
                ├─► 更新状态为 'running'
                │
                ├─► 添加用户消息
                │     └─► INSERT INTO cowork_messages
                │
                └─► CoworkRunner.startSession()
                      │
                      ├─► 创建 ActiveSession 对象
                      │     - abortController
                      │     - workspaceRoot
                      │     - confirmationMode
                      │
                      ├─► 组装 SystemPrompt
                      │     ├─► safetyPrompt (工作区安全规则)
                      │     ├─► localTimePrompt (本地时间)
                      │     ├─► userMemoriesXml (用户记忆)
                      │     ├─► memoryRecallPrompt (记忆策略)
                      │     └─► AGENT_SYSTEM_PROMPT.md (基础提示词)
                      │
                      └─► runClaudeCode()
                            │
                            ├─► 获取 API 配置
                            │
                            ├─► 加载技能文件
                            │
                            └─► 启动 Claude Code SDK
                                  │
                                  ├─► 流式输出消息 事件)
                                  │
                                  ├─► 权限请求 事件)
                                  │
                                  └─► 完成后触发 complete 事件
```

### 关键数据结构

**Session 对象**:
```typescript
{
  id: string;              // UUID
  title: string;           // 任务标题
  status: 'idle' | 'running';
  cwd: string;             // 工作目录
  systemPrompt: string;    // 系统提示词
  executionMode: 'local' | 'sandbox';
  activeSkillIds: string[]; // 启用的技能
  messages: CoworkMessage[];
}
```

**消息流转**:
- 用户消息 → 存入 SQLite → 触发 `message` 事件
- AI 响应 → 流式更新 → 触发 `messageUpdate` 事件
- 权限请求 → 弹窗确认 → `respondToPermission()` 返回结果

### SystemPrompt 组装详解

**源文件**: `sandbox/agent-runner/AGENT_SYSTEM_PROMPT.md`

**组装方法**: `CoworkRunner.composeEffectiveSystemPrompt()`

**最终 SystemPrompt 由 5 部分拼接**:

| 组件 | 来源 | 作用 |
|-----|------|------|
| `safetyPrompt` | 运行时生成 | 工作区边界限制，禁止越权操作 |
| `localTimePrompt` | 运行时生成 | 当前本地时间上下文 |
| `userMemoriesXml` | SQLite `user_memories` 表 | 用户偏好记忆 |
| `memoryRecallPrompt` | 硬编码 | 记忆检索策略说明 |
| `baseSystemPrompt` | AGENT_SYSTEM_PROMPT.md | AI 人格、行为规范、工具限制 |

> ⚠️ **注意**: `config.systemPrompt` 在 `coworkStore.ts:818` 中直接返回 `getDefaultSystemPrompt()`，即 `AGENT_SYSTEM_PROMPT.md` 文件内容。**用户无法通过 UI 修改此配置**，只能直接编辑该文件。

**前端组装流程** (`CoworkView.tsx:176-178`):
```typescript
const combinedSystemPrompt = [effectiveSkillPrompt, config.systemPrompt]
  .filter(p => p?.trim())
  .join('\n\n') || undefined;
```

| 组件 | 来源 | 说明 |
|-----|------|------|
| `effectiveSkillPrompt` | SkillService | 手动选择的技能提示词，或自动路由提示词 |
| `config.systemPrompt` | AGENT_SYSTEM_PROMPT.md | 全局基础提示词（不可 UI 配置） |

**AGENT_SYSTEM_PROMPT.md 核心内容**:
- **身份**: LobsterAI，网易有道开发的全场景个人助理
- **人格**: 协作型、工程质量导向、简洁友好
- **能力**: 数据分析、PPT 创建、视频生成、文档写作、邮件、定时任务
- **工具限制**: 禁用内置 WebSearch/WebFetch，通过技能实现网络功能
- **响应规范**: 保持用户语言一致、使用 `file://` 链接、适度更新频率

## 八、特色设计

1. **本地优先**: 所有配置存储在本地 SQLite，数据不上云
2. **多模型支持**: 内置国内外 10+ 模型提供商配置
3. **Coding Plan 模式**: Qwen/智谱/火山引擎/Moonshot 支持专属 Coding API 端点
4. **技能热更新**: 监听 SKILLs 目录变化，自动重载
5. **记忆系统**: 自动提取和存储用户偏好记忆
6. **IM 多平台**: 支持钉钉、飞书、Telegram、Discord、网易云信远程控制
