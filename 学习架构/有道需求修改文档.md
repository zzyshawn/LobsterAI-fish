# LobsterAI 需求修改文档

## 一、需求背景

用户希望能够自定义 AI 的人物设定，而不是使用系统默认的 `AGENT_SYSTEM_PROMPT.md`。当前系统提示词优先级为：系统默认 > 用户配置，需要调整为用户配置优先级更高。

## 二、修改目标

1. 创建会话目录时，自动在目录下新增 `人物设定.md` 文件
2. 该文件内容用于替换系统默认提示词
3. 提示词优先级调整为：**用户人物设定.md > 系统默认 AGENT_SYSTEM_PROMPT.md**
4. **支持实时修改**：用户编辑 `人物设定.md` 后，下一次对话立即生效，无需重启应用

## 三、技术方案

### 3.1 当前提示词加载流程

```
getDefaultSystemPrompt()
    └─► 读取 sandbox/agent-runner/AGENT_SYSTEM_PROMPT.md
```

### 3.2 修改后流程

```
创建会话目录
    │
    ├─► 创建目录
    │
    └─► 创建 人物设定.md (默认内容或空白)
    
加载提示词时
    │
    ├─► 检查工作目录下是否存在 人物设定.md
    │     ├─► 存在 → 读取用户设定
    │     └─► 不存在 → 使用系统默认
    │
    └─► composeEffectiveSystemPrompt() 组装最终提示词
```

### 3.3 核心修改点

1. **目录创建时** (`resolveTaskWorkingDirectory`)
   - 创建目录后，自动创建 `人物设定.md` 文件
   - **初始内容**: 复制系统默认 `AGENT_SYSTEM_PROMPT.md` 的内容
   - 用户可自行编辑修改

2. **提示词加载时** (`getDefaultSystemPrompt` 或新方法)
   - 优先检查工作目录下的 `人物设定.md`
   - 不存在时回退到系统默认
   - **每次对话时重新读取文件**，实现实时修改生效

3. **提示词组装时** (`composeEffectiveSystemPrompt`)
   - 用户提示词放在最后，优先级最高

### 3.4 人物设定.md 初始内容

文件创建时，使用系统默认提示词作为初始内容：

**来源文件**: `sandbox/agent-runner/AGENT_SYSTEM_PROMPT.md`

**内容摘要**:
- AI 身份：LobsterAI，网易有道开发的全场景个人助理
- 人格特征：专业、友好、高效
- 工具使用规范
- 响应格式要求

## 五、验收标准

1. 创建新会话时，工作目录下自动生成 `人物设定.md` 文件
2. 编辑 `人物设定.md` 后，新会话使用用户设定作为系统提示词
3. 删除 `人物设定.md` 后，回退使用系统默认提示词
4. 用户提示词内容优先级高于系统默认
5. **实时修改生效**：编辑保存文件后，下一次对话立即使用新内容，无需重启应用

## 六、风险评估

| 风险 | 影响 | 应对措施 |
|-----|------|---------||
| 文件编码问题 | 非UTF-8编码导致乱码 | 强制使用 UTF-8 读写 |
| 文件权限问题 | 无法创建文件 | 捕获异常，回退到系统默认 |
| 文件过大 | 影响性能 | 限制文件大小（如 10KB） |
| 用户删除文件 | 提示词丢失 | 回退到系统默认，不报错 |

## 八、实际修改内容

### 8.1 main.ts 修改

**文件**: `src/main/main.ts`

**修改位置**: `resolveTaskWorkingDirectory()` 函数

**修改内容**: 创建目录后，自动创建 `人物设定.md` 文件，初始内容为系统默认提示词

```typescript
const characterPromptPath = path.join(resolvedWorkspaceRoot, '人物设定.md');
if (!fs.existsSync(characterPromptPath)) {
  try {
    const defaultPromptPath = path.join(app.getAppPath(), 'sandbox', 'agent-runner', 'AGENT_SYSTEM_PROMPT.md');
    const defaultPromptContent = fs.readFileSync(defaultPromptPath, 'utf-8');
    fs.writeFileSync(characterPromptPath, defaultPromptContent, 'utf-8');
    console.log(`[Main] Created character prompt file: ${characterPromptPath}`);
  } catch (error) {
    console.warn('[Main] Failed to create character prompt file:', error);
  }
}
```

### 8.2 coworkRunner.ts 修改

**文件**: `src/main/libs/coworkRunner.ts`

**新增方法**: `getUserDefinedSystemPrompt(cwd: string)`

**功能**: 读取工作目录下的 `人物设定.md` 文件，实现实时修改生效

```typescript
private getUserDefinedSystemPrompt(cwd: string): string | null {
  const characterPromptPath = path.join(cwd, '人物设定.md');
  try {
    if (fs.existsSync(characterPromptPath)) {
      const content = fs.readFileSync(characterPromptPath, 'utf-8');
      const trimmed = content?.trim();
      if (trimmed) {
        console.log(`[CoworkRunner] Using user-defined system prompt from: ${characterPromptPath}`);
        return trimmed;
      }
    }
  } catch (error) {
    console.warn('[CoworkRunner] Failed to read user-defined system prompt:', error);
  }
  return null;
}
```

**修改位置**: `startSession()` 和 `continueSession()` 方法

**修改内容**: 优先使用用户定义的提示词

```typescript
const userDefinedPrompt = this.getUserDefinedSystemPrompt(sessionCwd);
const baseSystemPrompt = userDefinedPrompt ?? options.systemPrompt ?? session.systemPrompt;
```

### 8.3 提示词优先级

最终优先级（从高到低）：

1. **用户人物设定.md** (`工作目录/人物设定.md`) - 最高优先级
2. **options.systemPrompt** (技能提示词)
3. **session.systemPrompt** (会话存储的提示词)

**注意**: 未修改 `coworkStore.ts` 中的 `getDefaultSystemPrompt()`，保持其作为系统默认提示词的来源。
